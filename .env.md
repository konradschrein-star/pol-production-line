## Quick Setup

1. **Navigate to project folder:**
   ```

   ```

5. **Save the file**

6. **Start the system** with `START.bat`

## Required Variables

These variables MUST be set for the system to function.

### `DATABASE_URL`
**Purpose:** PostgreSQL database connection string

**Default Value:**
```bash
DATABASE_URL=postgresql://postgres:password@localhost:5432/obsidian_news_desk
```

**Format:**
```
postgresql://[username]:[password]@[host]:[port]/[database_name]
```

**When to Change:**
- Using external PostgreSQL server (not Docker)
- Changed Docker Compose database credentials

**DO NOT CHANGE if using default Docker setup.**

---

### `REDIS_URL`
**Purpose:** Redis connection string for BullMQ queues

**Default Value:**
```bash
REDIS_URL=redis://:obsidian_redis_password@localhost:6379
```

**Format:**
```
redis://:[password]@[host]:[port]
```

**When to Change:**
- Using external Redis server (not Docker)
- Changed Docker Compose Redis credentials

**DO NOT CHANGE if using default Docker setup.**

---

### `AI_PROVIDER`
**Purpose:** Selects which LLM service to use for script analysis

**Allowed Values:**
- `google` - Google AI (Gemini)
- `claude` - Anthropic Claude
- `groq` - Groq

**Recommendation:** Use `google` for free tier (60 requests/minute)

**Example:**
```bash
AI_PROVIDER=google
```

**Corresponding API Key Required:**
- If `AI_PROVIDER=google` → Must set `GOOGLE_AI_API_KEY`
- If `AI_PROVIDER=claude` → Must set `ANTHROPIC_API_KEY`
- If `AI_PROVIDER=groq` → Must set `GROQ_API_KEY`

---

**Purpose:** Chrome extension ID for Auto Whisk (Google Wisk automation)

**Default Value:**
```bash

```

**When to Change:**

- Auto Whisk extension gets updated with new ID

**DO NOT CHANGE unless instructed.**

---

### `LOCAL_STORAGE_ROOT`
**Purpose:** Local file system path for storing images, videos, and temp files

**Default Value:**
```bash
LOCAL_STORAGE_ROOT=C:\Users\konra\ObsidianNewsDesk
```

**Subdirectories (auto-created):**
- `images/` - Scene images from Google Wisk
- `avatars/` - HeyGen avatar MP4 files
- `videos/` - Final rendered broadcast videos
- `temp/` - Temporary processing files

**When to Change:**
- Using a different drive (e.g., D:\ for more space)
- Want to organize files differently

**Example:**
```bash
LOCAL_STORAGE_ROOT=D:\Obsidian\Broadcasts
```

**Requirements:**
- Must be an absolute path
- User must have write permissions
- At least 50GB free space recommended

## Optional Variables

These variables have defaults and can be omitted.

### `REMOTION_CONCURRENCY`
**Purpose:** Number of parallel rendering threads for Remotion

**Default:** `2` (uses 2 CPU cores)

**Example:**
```bash
REMOTION_CONCURRENCY=8
```

**When to Increase:**
- High-end CPU with 12+ cores
- Want faster rendering (uses more CPU)

**When to Decrease:**
- Low-end CPU (4 cores or fewer)
- Rendering causes system lag

---

### `REMOTION_QUALITY`
**Purpose:** Video quality preset for Remotion rendering

**Allowed Values:**
- `low` - Faster rendering, lower quality (720p)
- `medium` - Balanced (1080p, standard bitrate)
- `high` - Slower rendering, best quality (1080p, high bitrate)

**Default:** `high`

**Example:**
```bash
REMOTION_QUALITY=medium
```

**When to Change:**
- Need faster previews (use `low`)
- Publishing to social media only (use `medium`)

---

### `IMAGE_GENERATION_DELAY`
**Purpose:** Delay (in milliseconds) between image generation requests to avoid bans

**Default:** `60000` (60 seconds)

**Example:**
```bash
IMAGE_GENERATION_DELAY=90000
```

**When to Increase:**
- Frequently triggering Google Wisk ban detection
- Want to be extra safe (use 90-120 seconds)

**When to Decrease:**
- Using paid Google Wisk account with higher limits
- Testing in development (not recommended for production)

**⚠️ WARNING:** Values below 30 seconds will likely trigger bans.

---

### `MAX_RETRIES`
**Purpose:** Number of times to retry failed scene image generation

**Default:** `3`

**Example:**
```bash
MAX_RETRIES=5
```

**When to Increase:**
- Network is unstable
- Google Wisk occasionally fails

**When to Decrease:**
- Want to fail fast (manual intervention preferred)

---

### `NODE_ENV`
**Purpose:** Node.js environment mode

**Allowed Values:**
- `development` - Shows detailed errors, hot reload enabled
- `production` - Optimized builds, error details hidden

**Default:** `development` (set by Next.js)

**Example:**
```bash
NODE_ENV=production
```

**⚠️ NOTE:** In production mode, you must run `npm run build` before `npm start`.

## Getting API Keys

### Google AI (Recommended)

**Provider:** Google AI Studio
**Cost:** Free tier (60 req/min)
**Model:** Gemini 1.5

**Steps:**

1. **Visit Google AI Studio:**
   ```
   https://aistudio.google.com/app/apikey
   ```

2. **Sign in with Google account** (any Gmail address)

3. **Click "Get API Key" button**

4. **Create API Key:**
   - Click "Create API Key"
   - Select a Google Cloud project (or create new one)
   - Copy the generated key (starts with `AIzaSy`)

5. **Paste into `.env` file:**
   ```bash
   AI_PROVIDER=google
   GOOGLE_AI_API_KEY=AIzaSyD1234567890abcdefghijklmnopqrstu
   ```

6. **Save `.env` file**

**Troubleshooting:**
- If "Get API Key" is disabled, enable the Gemini API in Google Cloud Console
- If you get quota errors, you've exceeded free tier limits (wait 24 hours or upgrade)

---

### Anthropic Claude

**Provider:** Anthropic
**Cost:** Pay-as-you-go (no free tier)
**Model:** Claude 3.5 Sonnet

**Steps:**

1. **Visit Anthropic Console:**
   ```
   https://console.anthropic.com/
   ```

2. **Sign up for an account** (requires email and payment method)

3. **Navigate to "API Keys" section**

4. **Click "Create Key" button**
   - Give key a name (e.g., "Obsidian News Desk")
   - Copy the generated key (starts with `sk-ant-api03-`)

5. **Paste into `.env` file:**
   ```bash
   AI_PROVIDER=claude
   ANTHROPIC_API_KEY=sk-ant-api03-1234567890abcdefghijklmnopqrstuvwxyz
   ```

6. **Save `.env` file**

**Pricing:**
- ~$3 per 1M input tokens
- ~$15 per 1M output tokens
- Typical broadcast costs ~$0.05-0.10 per analysis

**Troubleshooting:**
- If API key doesn't work, check billing is enabled
- Check API quota in Anthropic Console

---

### Groq

**Provider:** Groq
**Cost:** Free tier (30 req/min)
**Model:** Llama 3.1 70B

**Steps:**

1. **Visit Groq Console:**
   ```
   https://console.groq.com/keys
   ```

2. **Sign up for an account** (email required, no payment method)

3. **Click "Create API Key" button**
   - Give key a name (e.g., "Obsidian News Desk")
   - Copy the generated key (starts with `gsk_`)

4. **Paste into `.env` file:**
   ```bash
   AI_PROVIDER=groq
   GROQ_API_KEY=gsk_1234567890abcdefghijklmnopqrstuvwxyz
   ```

5. **Save `.env` file**

**Free Tier Limits:**
- 30 requests per minute
- 14,400 requests per day
- Sufficient for ~50 broadcasts per day

**Troubleshooting:**

## Troubleshooting

### Error: "Missing DATABASE_URL"

**Cause:** `.env` file doesn't exist or doesn't have `DATABASE_URL`

**Fix:**
1. Verify `.env` file exists in project root
2. Open `.env` in text editor
3. Add line:
   ```bash
   DATABASE_URL=postgresql://postgres:password@localhost:5432/obsidian_news_desk
   ```
4. Save and restart system

---

### Error: "Invalid AI_PROVIDER"

**Cause:** `AI_PROVIDER` is set to an invalid value

**Valid Values:** `google`, `claude`, `groq`

**Fix:**
1. Open `.env` file
2. Change `AI_PROVIDER` to one of the valid values:
   ```bash
   AI_PROVIDER=google
   ```
3. Save and restart system

---

### Error: "Missing [AI_PROVIDER] API Key"

**Cause:** You set `AI_PROVIDER=google` but didn't provide `GOOGLE_AI_API_KEY`

**Fix:**
1. Get API key from corresponding provider (see [Getting API Keys](#getting-api-keys))
2. Add to `.env` file:
   ```bash
   GOOGLE_AI_API_KEY=your_actual_key_here
   ```
3. Save and restart system

---

### Error: "API key is invalid"

**Cause:** API key is incorrect, expired, or quota exceeded

**Fix:**
1. **Check key is correct:**
   - Copy-paste carefully (no extra spaces)
   - Google AI keys start with `AIzaSy`
   - Claude keys start with `sk-ant-api03-`
   - Groq keys start with `gsk_`

2. **Regenerate key:**
   - Go to provider console (see [Getting API Keys](#getting-api-keys))
   - Delete old key
   - Create new key
   - Update `.env` file

3. **Check quota:**
   - Google AI: Check https://aistudio.google.com/app/apikey → Quota
   - Claude: Check Anthropic Console → Usage
   - Groq: Check Groq Console → Usage

---

### Error: "Could not connect to database"

**Cause:** PostgreSQL Docker container not running

**Fix:**
1. **Check Docker Desktop is running:**
   - Open Docker Desktop application
   - Ensure whale icon is active in system tray

2. **Start database manually:**
   ```bash
   docker-compose up -d postgres
   ```

3. **Check database logs:**
   ```bash
   docker-compose logs postgres
   ```

4. **Restart entire system:**
   - Run `STOP.bat`
   - Wait 10 seconds
   - Run `START.bat`

---

### Error: "Could not connect to Redis"

**Cause:** Redis Docker container not running

**Fix:**
1. **Check Docker Desktop is running**

2. **Start Redis manually:**
   ```bash
   docker-compose up -d redis
   ```

3. **Check Redis logs:**
   ```bash
   docker-compose logs redis
   ```

4. **Restart entire system:**
   - Run `STOP.bat`
   - Wait 10 seconds
   - Run `START.bat`

---

### Error: "LOCAL_STORAGE_ROOT does not exist or is not writable"

**Cause:** Path doesn't exist or user lacks permissions

**Fix:**
1. **Create directory manually:**
   - Open File Explorer
   - Navigate to `C:\Users\konra\`
   - Right-click → New → Folder
   - Name it `ObsidianNewsDesk`

2. **Check permissions:**
   - Right-click folder → Properties → Security
   - Ensure your user account has "Full Control"

3. **Use different path:**
   - Edit `.env` file:
     ```bash
     LOCAL_STORAGE_ROOT=C:\Users\konra\Documents\Obsidian
     ```
   - Save and restart system

---

### Error: "AUTO_WHISK_EXTENSION_ID is invalid"

**Cause:** Extension ID is malformed or incorrect

**Fix:**
1. **Reset to default:**
   ```bash
   AUTO_WHISK_EXTENSION_ID=gcgblhgncmhjchllkcpcneeibddhmbbe
   ```

2. **Check extension is installed:**
   - Open Chrome
   - Navigate to `chrome://extensions/`
   - Find "Auto Whisk" extension
   - Copy ID from extension card

3. **Update `.env` with correct ID**
